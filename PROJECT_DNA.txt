# PROJECT DNA: NEXLED UI SYSTEM
==================================================

This document serves as the master guide for the NexLed project. It contains the technical, design, and structural rules required for any developer or AI to maintain consistency while building new features.

---

## 1. PROJECT OVERVIEW
NexLed is a premium eCommerce platform specializing in professional LED lighting solutions (Interior, Retail, Industrial). The goal is a high-end, professional, and visually "stunning" user experience.

- **Frontend Strategy**: Rapid drafting using Tailwind CSS via CDN with a custom configuration file (`config-cdn.js`).
- **Backend Strategy**: PHP MVC-inspired architecture for production scalability.

---

## 2. DESIGN SYSTEM & VISUAL RULES

### A. Color Palette
- **Green Primary**: `#00813D` (Main brand color, CTAs)
- **Green Secondary**: `#19231D` (Darker accents, status)
- **Grey Primary**: `#595959` (Text descriptions, accents)
- **Grey Tertiary**: `#F6F6F6` (Main background color)
- **White**: `#FFFFFF` (Surface color for cards/sections)
- **Black**: `#000000` (Main headings, dark mode elements)

### B. Typography
- **Main Font**: 'Urbanist' (Sans-serif) via Google Fonts.
- **Weights**: 
  - Titles/Headings: Bold (700) or Medium (500).
  - Body/Descriptions: Light (300) or ExtraLight (200).
- **Scale**: Use Tailwind text scales (text-3xl, text-6xl, etc.) with `tracking-tight` for large headings.

### C. Geometry & Aesthetics
- **Rounded Corners**: 
  - Main containers/Sections: `32px` (Tailwind: `rounded-[32px]`)
  - Product Cards/Small cards: `24px` (Tailwind: `rounded-[24px]`)
  - Buttons/Small inputs: `12px` or `16px` (Tailwind: `rounded-xl` or `rounded-2xl`)
- **Effects**:
  - **Glassmorphism**: Use white backgrounds with opacity and blur (`backdrop-blur-sm`).
  - **Shadows**: Premium, subtle shadows (Tailwind: `shadow-[0_20px_40px_rgba(0,0,0,0.05)]`).
  - **Transitions**: Use `cubic-bezier(0.16, 1, 0.3, 1)` for high-end "snappy yet smooth" animations.

## 3. PROJECT ARCHITECTURE

### A. Folder Structure
- `/public`: Web root. Contains `index.php` (router) and static `assets`.
- `/app`: Backend logic.
  - `/config`: DB and App settings.
  - `/models`: Database classes (Product, Category).
  - `/views`: PHP templates and partials.
- `/src`: Development source.
  - `/pages`: Flat list of .html prototypes (recently reorganized).
  - `/components`: Atomic Design units (atoms, molecules, organisms).
  - `/tokens`: Shared JS design definitions.

### B. Navigation & Header
Every page MUST use the **Unified Card Header** containing:
1. Announcement Bar (Dynamic closure).
2. Navbar (Transparent on colored BG or white on light BG).
3. Integrated Logo and Shopping Cart.

---

## 4. GUIDELINES FOR NEW FEATURES

1. **Atomic Consistency**: Before creating a new component, check if an existing Atom or Molecule can be reused.
2. **Mobile-First**: Always verify how a grid or flex container behaves on small screens.
3. **Naming Convention**:
   - Files: Use kebab-case (`product-card.html`).
   - Classes: Prefer Tailwind utility classes over custom CSS.
4. **Interactive States**: Every interactive element (Button, Card, Link) MUST have a hover state, usually involving a slight shadow increase, scale adjustment, or color shift.
5. **PHP Integration**:
   - Never use raw SQL; always use PDO with prepared statements.
   - Sanitize all user-facing output using `e()` or `htmlspecialchars()`.
   - Keep controllers slim and logic inside Models/Includes.

---

## 5. EXTERNAL LIBRARIES
- **Icons**: RemixIcon (CDN) - `ri-` prefix.
- **Styling**: Tailwind CSS (CDN - v3.4+).
- **Functionality**: Vanilla JavaScript (Avoid jQuery unless required for specific plugins).
